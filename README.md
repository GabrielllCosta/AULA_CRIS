   O model basicamente trata da lódica de banco de dados, acredito que a lógica é criada em userModel.js, e para cada tabela existente é necessário um arquivo .js, e o model não cuida de nada do que vai ser passado na tela, apenas mexe com o banco de dados até a agora.
   O controller é caminho que atraves das requisições organiza os acessos do banco de dados.
    Endpoints são caminhos url que conecta o back e  front, exportando as informações.
